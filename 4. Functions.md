Functions in Go are defined using the `func` keyword. Go allows functions to return **multiple values**, which is a powerful feature for error handling and simplifying code.

### 1. Function Basics

The structure is simple: `func` keyword, function name, parameters (with types), and the return type.

#### Example: Simple Function

Go

```
package main

import "fmt"

// add takes two integers (a and b) and returns an integer
func add(a int, b int) int {
    return a + b
}

func main() {
    result := add(5, 7)
    fmt.Println("5 + 7 =", result) // Output: 5 + 7 = 12
}
```

**ğŸ’¡ Note on Parameters:** If parameters are of the same type, you only need to specify the type once for the last parameter: `func add(a, b int) int`.

### 2. Multiple Return Values

This is where Go shines! A function can return two or more values, often used to return a result **and** an error status.

#### Example: Calculating Area with an Error

Go

```
package main

import (
    "fmt"
    "errors" // Package for creating simple errors
)

// calcArea returns the area (float64) and an error (error type)
func calcArea(width, height float64) (float64, error) {
    if width <= 0 || height <= 0 {
        // Return 0 and a new error message
        return 0, errors.New("dimensions must be positive")
    }
    // Return the calculated area and nil (meaning no error)
    return width * height, nil
}

func main() {
    // 1. Success case
    area, err := calcArea(10, 5)
    
    // Always check the error value first!
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Calculated Area:", area) // Output: Calculated Area: 50
    }

    // 2. Failure case
    _, err = calcArea(-2, 5) // Use '_' to discard the 'area' result
    
    if err != nil {
        fmt.Println("Error:", err) // Output: Error: dimensions must be positive
    }
}
```

### 3. Named Return Values (Naked Returns)

You can name the return variables in the function signature. This can improve clarity, especially with multiple returns, and allows for a "naked" `return` statement that automatically returns the named variables.

Go

```
// The return values 'sum' and 'diff' are declared here
func operation(a, b int) (sum int, diff int) {
    sum = a + b
    diff = a - b
    // A 'naked' return uses the named variables (sum and diff)
    return 
}
```
